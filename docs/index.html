See what your audience actually sees.

A single-file Python tool that reads every web map in an ArcGIS Online organisation and finds the layers their viewers cannot load.

For every layer reference it answers one question: is every viewer of the map also a viewer of the layer?

A public map with an org-only layer shows the public an empty canvas.

Nobody gets told. The map still opens, the legend still lists the layer, and the tiles never arrive. The owner sees it working, because the owner can see the layer.

Sharing drifts one item at a time. A layer moves to a group, a service is republished under a new name, an item is deleted while three maps still point at it. Each change is invisible from the map that depends on it.

sightline reads the whole organisation and reports every one of those, ranked by how many people are looking.

Read-only

It changes nothing in your organisation. No writes, no re-sharing, no item edits.

Run it as an administrator

A standard user only sees their own content, so the audit silently covers a fraction of the org and everything else reads as clean.

One file

sightline.py. No package, no framework, and no dependency beyond the three ArcGIS Pro already ships.

Verify before you trust it

--self-check runs the full assertion suite with no network and no credentials. It asserts the classification matrix, so a regression fails loudly instead of quietly under-reporting.

Smoke-test first

--limit 10 caps the scan while you confirm the connection and the output path.

Credentials

Resolved in order: the config block, then ARCGIS_USER / ARCGIS_PASS, then a signed-in ArcGIS Pro session, then an interactive prompt. The script writes no password to disk.

Configure

Everything tunable sits in one CONFIG block at the top of the file. Command-line flags override the matching value.

Do not put a real password in PASSWORD. A password typed into a script is a password you will eventually commit. The field exists for locked-down automation where environment variables are not available.

Reading the report

Three sheets. Findings is everything that is not OK, worst first. This is the work queue. AllLayers is every reference examined, including healthy ones, for pivoting. Summary carries run metadata, verdict totals, the worst maps, and the owners with the most problems. That last table is who you email.

VerdictMeaningTypical cause
OKEvery map viewer can load the layer
BROKENSome map viewers cannot load the layerPublic map with an org-only layer; a token-secured service in a public map
DEADThe item or service no longer existsItem deleted; service removed or renamed
UNKNOWNCould not be determinedUnreachable host, or group membership unavailable. The Reason column says which

Severity follows the map, not the layer

A broken layer in a private map is a note to self. The same layer in a public map is on your website right now. Bands run 1 High public, 2 Medium org, 3 Low shared, 4 Info private. Findings sorts by severity then view count, so the most-visited broken public map is the first row you read.

The Overexposed column

A separate problem, deliberately kept out of Findings: a public layer inside a non-public map. Nothing is broken, but the data is shared more widely than its map implies. Only org-owned layers are flagged, so basemaps create no noise.

Errors are findings

A map that fails to read becomes a row with its exception recorded, not a silent omission.

How it decides

Sharing is audience containment, not a severity rank. The common shortcut, private < org = shared < public, hides real breakage.

org and shared are incomparable audiences, not equal ones. An org member outside the group cannot load a group-shared layer, so an org-wide map with a group-shared layer is broken for most of its audience even though a rank comparison calls the two equivalent.

LayerMapResultWhy
publicanythingOKPublic contains every audience
anythingprivateOKThe audience is the owner alone
private (other owner)privateBROKENNot even the map’s owner can load it
orgpublicBROKENAnonymous visitors are not org members
orgorg / sharedOKGroup members are org members
sharedorg / publicBROKENThe audience is wider than the group

sharedshared, groups unknownUNKNOWNNever guessed

A map shared to zero groups normalises to private, since its real audience is the owner.

The layers that are not items at all

In the organisation this was built against, 57% of layer references are not ArcGIS Online items at all. They are registered service URLs, on-premises servers and basemaps. Tools that only compare item permissions mark every one of them fine.

sightline fetches each distinct service URL anonymously, with no token, and reads what the public would get. A service answering 499 inside a public map means every anonymous visitor gets an empty map. TLS verification stays on, because a certificate a browser would reject is a finding rather than something to suppress.

anonymous probe → verdict
responds normally      → the layer really is public
401 403 498 499        → needs a token
                          in a public map: BROKEN
400 404                → DEAD, the service is gone
connection failure     → UNKNOWN, reason recorded

Why it is fast

The naive version of this audit takes about 19 minutes against a 336-map organisation. This one takes under three. Three things account for the difference, each measured.

0.665 s

Never touch an unknown attribute on an Item. Item.__getattr__ performs a network round trip before raising AttributeError. One hasattr() call costs this much. The tool sightline replaced called it twice per layer to fill two columns that were empty in 171 of 171 rows, spending roughly 95% of its runtime in dead code.

18.8x

Resolve items in batches. One id:(a OR b OR ...) search resolves 50 ids per round trip instead of one, measured 18.8 times faster than calling content.get() per id with byte-identical results.

0.7x

Do not reach for threads. On a shared GIS session, 4 workers gave 1.3x, 8 gave 1.1x and 16 gave 0.7x, slower than sequential. The connection serialises, so the script stays sequential and readable.

What it does not do

Web maps only

Dashboards, Experience Builder apps, StoryMaps and Web Mapping Applications are not scanned. In a typical organisation those outnumber web maps roughly two to one, so this is a real gap.

Group membership is item-level

It compares which groups a map and a layer are shared to. It does not enumerate group members, so it cannot tell you a group is empty.

UNKNOWN is honest, not lazy

An unreachable host may be a firewall rule rather than a defect. Check the Reason column before acting.

A secured service in a non-public map stays UNKNOWN

Whether your org members hold credentials for an on-premises service cannot be determined from ArcGIS Online.
